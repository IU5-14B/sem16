Семинар 16: Алгоритмы обработки массивов

Обзор проекта

Проект включает реализацию трех задач по обработке массивов целых чисел:

  • Задача 1: Поиск медианы через сортировку
  • Задача 3: Нахождение пересечения двух множеств
  • Задача 4: Поиск минимального и максимального элемента

Все алгоритмы реализованы на C++ и организованы в следующие файлы:

  • algorithms.hpp - заголовочный файл с объявлениями
  • algorithms.cpp - реализации алгоритмов
  • main.cpp - интерактивная программа для тестирования

Структура проекта

algorithms.hpp

Заголовочный файл содержит объявления всех функций и структур:

  • median_sort() - медиана через сортировку
  • intersection_bruteforce() - пересечение "грубой силой"
  • intersection_sort_two_pointers() - пересечение через два указателя
  • minmax_bruteforce(), minmax_sort(), minmax_decomposition() - три алгоритма поиска min/max

main.cpp

Интерактивная программа с меню для тестирования алгоритмов:

  • Ввод массивов вручную или генерация случайных
  • Меню для выбора задачи
  • Вывод результатов с информацией о сложности

Задача 1: Медиана через сортировку

Описание: Функция находит медиану массива как элемент с индексом n/2 после сортировки.

Сложность: O(n log n)

Реализация:

int median_sort(std::vector<int> v) { if (v.empty()) { throw std::invalid_argument("median_sort: empty array"); } std::sort(v.begin(), v.end()); return v[v.size() / 2]; }

Особенности:

  • Принимает копию массива, чтобы не изменять исходные данные
  • Использует стандартную функцию сортировки std::sort
  • Для нечетного размера массива возвращает средний элемент, для четного - элемент с индексом n/2

Задача 3: Пересечение множеств

Описание: Нахождение пересечения двух множеств (массивов) целых чисел.

Алгоритм 1: "Грубая сила" (Brute Force)

Сложность: O(n*m)

Проверяет каждый элемент первого массива на наличие во втором массиве.

std::vector<int> intersection_bruteforce(const std::vector<int>& a, const std::vector<int>& b) { std::vector<int> result; for (int x : a) { for (int y : b) { if (x == y) { if (!contains(result, x)) { result.push_back(x); } break; } } } return result; }

Алгоритм 2: Сортировка + два указателя

Сложность: O(n log n + m log m)

Сортирует оба массива и использует два указателя для нахождения общих элементов.

std::vector<int> intersection_sort_two_pointers(std::vector<int> a, std::vector<int> b) { std::sort(a.begin(), a.end()); std::sort(b.begin(), b.end()); std::vector<int> result; std::size_t i = 0, j = 0; while (i < a.size() && j < b.size()) { if (a[i] < b[j]) { ++i; } else if (b[j] < a[i]) { ++j; } else { // a[i] == b[j] int value = a[i]; if (result.empty() || result.back() != value) { result.push_back(value); } ++i; ++j; } } return result; }

Задача 4: Минимум и максимум

Описание: Поиск минимального и максимального элемента в массиве тремя способами.

Алгоритм Сложность Описание
minmax_bruteforce
O(n)
 Один проход по массиву, сравнение каждого элемента с текущими min и max
minmax_sort
O(n log n)
 Сортировка массива, затем первый и последний элементы
minmax_decomposition
O(n)
 Попарное сравнение элементов, затем поиск min среди локальных минимумов и max среди локальных максимумов

Алгоритм декомпозиции (попарное сравнение):

MinMax minmax_decomposition(const std::vector<int>& v) { if (v.empty()) throw std::invalid_argument(...); if (v.size() == 1) return {v[0], v[0]}; std::vector<int> local_mins, local_maxs; std::size_t i = 0; while (i + 1 < v.size()) { int a = v[i], b = v[i + 1]; if (a < b) { local_mins.push_back(a); local_maxs.push_back(b); } else { local_mins.push_back(b); local_maxs.push_back(a); } i += 2; } // если нечётное количество элементов if (i < v.size()) { local_mins.push_back(v[i]); local_maxs.push_back(v[i]); } // поиск минимума среди local_mins int mn = local_mins[0]; for (std::size_t k = 1; k < local_mins.size(); ++k) { if (local_mins[k] < mn) mn = local_mins[k]; } // поиск максимума среди local_maxs int mx = local_maxs[0]; for (std::size_t k = 1; k < local_maxs.size(); ++k) { if (local_maxs[k] > mx) mx = local_maxs[k]; } return {mn, mx}; }

Сравнение алгоритмов min/max:

  • Грубая сила и декомпозиция работают за O(n), но декомпозиция выполняет примерно 3n/2 сравнений против 2n у грубой силы
  • Сортировка медленнее по асимптотике, но после сортировки могут быть полезны другие операции
  • Декомпозиция более эффективна по количеству сравнений, но требует дополнительной памяти для хранения локальных минимумов и максимумов

Особенности реализации

Обработка ошибок:

  • Все функции проверяют пустые входные массивы и выбрасывают исключения
  • В main.cpp используется блок try-catch для обработки исключений

Устранение дубликатов:

  • В алгоритмах пересечения обеспечивается уникальность элементов в результате
  • Используется проверка result.back() != value и пропуск дубликатов

Интерактивный интерфейс:

  • Поддержка ручного ввода и случайной генерации массивов
  • Валидация входных данных (размеры, диапазоны)
  • Четкое меню с пояснениями алгоритмов и их сложности
